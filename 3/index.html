Vocabulary · Use These Words Today (3 min)

The 6 key words 🔑

🎯 Recommender

A piece of code whose only job is to pick what to show next out of many choices.

⭐ Rating

A number that describes how much someone liked an item. Today: 1 = liked, 0 = didn't.

📊 Average

Add up all the ratings, divide by how many. A summary of "what most people thought."

🥇 Ranking

A list ordered from best to worst. The output of every recommender is a ranking.

🧍 Personalization

Showing each person a different ranking based on their own taste — not the crowd's.

Quick Check

If 5 friends each rate "Sports" with 1, 1, 1, 1, 1, what is the average — and what does that tell us?

Algorithm 1 · Average Rating (4 min)

The simplest rule that works ⭐

The rule, in plain English

For every game, add up all the thumbs-ups, divide by how many people voted. Whichever game has the highest average wins the top spot.

# pseudo-code for game in ratings: score = sum(votes) / len(votes) recommend max(score)

?Predict before we run it

  • Puzzle → [1,0,0,1,0]
  • Shooter → [0,0,1,0,0]
  • Sports → [1,1,1,1,1]

Live Code · Type Along (8 min)

Average-Rating Recommender 💻

Recommender_v1.py

ratings = { 'Racing': [1, 1, 0, 1, 1], 'Puzzle': [1, 0, 0, 1, 0], 'Shooter': [0, 0, 1, 0, 0], 'Sports': [1, 1, 1, 1, 1], } averages = {} for game, votes in ratings.items(): score = sum(votes) / len(votes) averages[game] = score print('Average scores:') for game, score in averages.items(): print(game, '->', round(score, 2)) best_game = max(averages, key=averages.get) print('Recommended first:', best_game)

?Read the code line by line

  • What does .items() hand back to us each loop?
  • Why divide by len(votes) and not by 5?
  • What does key=averages.get tell max() to do?
  • Predict the printed winner — then run it. Were you right?

Critique · Where the Rule Breaks (4 min)

When does "most popular" go wrong? 🚧

🎂

Tastes differ

Most kids in class love Sports — but you love Puzzle. The popular pick isn't your pick.

🆕

Cold start

A brand-new game has zero votes. Average = ??? It can never appear at the top — even if it's amazing.

📣

Rich-get-richer

Whatever shows up first gets more clicks → higher average → shown more. The leader stays the leader.

🧠 Class debate: Imagine a librarian who only recommends the most-borrowed book. What books would never reach a reader? Whose voice goes missing?

Live Code · Type Along (10 min)

Per-Player Recommender 🧍🧍

Recommender_v2.py

player_ratings = { 'Alex': {'Racing':[1,1,1], 'Puzzle':[1,1,0], 'Shooter':[0,0,0]}, 'Jordan': {'Racing':[0,1,0], 'Puzzle':[1,1,1], 'Shooter':[1,1,0]}, } for name, games in player_ratings.items(): print('\nRecommendations for', name) averages = {} for game, votes in games.items(): averages[game] = sum(votes) / len(votes) ranked = sorted(averages.items(), key=lambda item: item[1], reverse=True) for game, score in ranked: print(' ', game, '->', round(score, 2))

?Trace the loops

  • The outer loop runs 2 times — why? The inner loop runs 3 times — why?
  • sorted(... reverse=True) — what does that flip?
  • lambda item: item[1] — why item 1 and not 0?
  • Predict: who gets Racing first — Alex or Jordan?

Compare · Two Tools, Two Jobs (3 min)

Average vs. Personalized — when to use each ⚖️

📊 Average rating

  • ✅ Fast and easy to explain
  • ✅ Great for "trending" or "popular now"
  • ❌ Ignores personal taste
  • ❌ Suffers from cold start

Use when: you have many users but very little data per user.

🧍 Per-user (personalized)

  • ✅ Captures individual taste
  • ✅ Improves the more you watch
  • ❌ Needs lots of data per user
  • ❌ Can build filter bubbles

Use when: each user has plenty of history and you want a unique homepage.

🤝 In real apps (Netflix, YouTube, Spotify) the two are combined: average for new users, personalized for regulars, plus a sprinkle of randomness. Which approach is closest to your intuition?
