📚 Warm-Up · Part B
Quick Checks — 20 Questions

Click the best answer for each question — your choice highlights in blue. These 20 mirror the multiple-choice style of the real test and cover every topic twice. Try it first — the answer key is at the end.

1Algorithms Session 1–2

A computer follows instructions best when they are…

A clear and step-by-step
B secret and surprising
C mostly guessed
D as short as possible, even if steps are missing
Remember the sandwich robot: if you skip a step, the robot does the wrong thing. Computers do exactly what you tell them — no guessing.
2Safety & Responsible AI Session 1–2

If an AI gives you a strange or wrong answer, you should…

A believe it no matter what
B check it yourself — AI can be wrong
C share your password to fix it
D turn off the computer
🔐 Two golden rules: AI can be wrong — always check, and never share private info (name, address, passwords).
3Scratch Events Session 3

In Scratch, the when 🏁 clicked block usually…

A starts a project
B stops all sound
C deletes a sprite
D changes color
An event is what starts code — a green-flag click or a key press. A sprite is the character you program.
4Loops Session 4

A loop is used to…

A repeat instructions
B delete code
C rename a sprite
D turn off the computer
🔄 A loop saves you from copying blocks over and over. repeat 4 draws a square; forever never stops.
5Conditionals Session 5

An if block lets a program…

A make a decision
B repeat forever
C print a picture
D save a file
❓ A conditional chooses based on true/false. “If touching edge, then bounce.” This is the seed of an AI classifier (is it a cat?).
6Variables Session 6

A variable is like…

A a labeled box that stores a value
B a permanent number that never changes
C a drawing tool
D a sound effect
📦 Variables hold score, lives, or a name — a value that can change. AI keeps a variable too: its confidence score.
7Projects & Debugging Session 7

A project is “shippable” when…

A it is perfect with zero mistakes
B it starts, responds to the player, and ends or loops
C it uses the most blocks possible
D only the teacher can run it
🚀 The 3 shippable rules: It STARTS · It RESPONDS · It ENDS or LOOPS. A bug is a normal, fixable problem.
8Python Session 8

In Python, print("Hello") will…

A show the word Hello on the screen
B print a picture of hello
C do nothing
D ask a question
🐍 Scratch and Python are the same brain, different voice. Scratch say [Hello] = Python print("Hello"). Don’t forget the quotes and parentheses!
9AI Concepts All Sessions

Training data is…

A examples an AI learns from
B a password
C a kind of loop
D a robot arm
🤖 The AI recipe: training data → model → prediction. More and better examples = better guesses. Few examples = more mistakes.
10Mixed Skills All Sessions

Which list is in the right order from concrete to text coding?

A Python first, then Scratch, then algorithms
B Algorithm idea → Scratch blocks → Python text
C Only Scratch, never Python
D Loops before we learn what an algorithm is
🎗 We built up: plan the steps (algorithm) → snap the blocks (Scratch) → type the same logic (Python). Same ideas, three ways.
11Algorithms Session 1–2

You want a robot to open a door. Which instruction is most useful?

A “Do the door thing”
B Reach to the handle, turn it, then pull the door open
C “Be smart”
D “Open it somehow”
🧠 The more specific and step-by-step your instructions, the better a computer or robot can follow them.
12Safety & Responsible AI Session 1–2

A good digital citizen…

A shares their home address with any app that asks
B keeps private info private and asks a grown-up when unsure
C believes everything a chatbot says
D uses other people’s passwords
🔐 Keep your name, address, and passwords private. When something feels wrong online, ask a trusted grown-up.
13Scratch Events Session 3

A sprite in Scratch is…

A a character or object you program
B a type of loop
C the same as the green flag
D a way to save your project
🎭 Sprites are the actors on your stage. Events (like a key press) tell them when to act.
14Loops Session 4

To draw a square with a turtle or sprite, the best loop is…

A repeat 4 (turn + move each time)
B repeat 100
C no loop at all
D a forever loop that never ends
🔄 A square has 4 equal sides, so repeat 4 is perfect. Loops mean you don’t repeat yourself!
15Conditionals Session 5

What is the difference between if and if-else?

A if-else gives a second path when the answer is false
B they are exactly the same
C if repeats forever
D if-else is only for Python
❓ if = do something when true. if-else = do one thing when true, another thing when false.
16Variables Session 6

When an AI makes a guess, the “confidence” number it keeps is most like a…

A variable (a value that changes)
B loop
C sprite
D green flag
📦 A variable stores something that can change — a game score, or an AI’s confidence (“I’m 80% sure it’s a cat”).
17Projects & Debugging Session 7

Your Scratch game does nothing when you click the green flag. The most likely fix is…

A connect your code to a when 🏁 clicked block
B delete the whole project
C add more sprites
D change the backdrop
🚀 No start event = code never runs. The green flag is your project’s “on” switch. Finding this is debugging!
18Python Session 8

What does for i in range(3): do?

A repeats the indented code 3 times
B prints the number 3
C asks the user 3 questions
D makes a variable called range
🐍 for is Python’s loop. range(3) counts 0, 1, 2 — that’s 3 times. Same idea as Scratch repeat 3.
19AI Concepts All Sessions

Why might an AI trained on only 3 pictures make more mistakes?

A it hasn’t seen enough examples to spot the pattern
B 3 is an unlucky number
C more pictures always make it worse
D AI never makes mistakes
🤖 More and clearer training data = better predictions. Too few examples → the model guesses wrong more often.
20Mixed Skills All Sessions

A chatbot works by following this pattern:

A input → process → output
B output → input → delete
C it just guesses with no rules
D it only works in Scratch
💬 A chatbot takes your words (input), decides what to do (process), and replies (output) — just like input() then print().
