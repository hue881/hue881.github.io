💡 Mini-Lesson

What is a function, really?

A function is a named block of code that runs only when you call it — like a recipe you can follow anytime.

Think of it like a vending machine:

  • You press the button (call the function)
  • The machine does the same steps every time
  • It gives you something back (a return value)

💡 Mini-Lesson · Syntax

Two moves: define, then call

# 1. DEFINE the function (the recipe)
def greet(name):
    print("Hello,", name, "!")

# 2. CALL the function (press the button)
greet("Maya")
greet("Leo")

Question to hold onto:

If we call greet("Maya") and then greet("Leo"), does the function's code change at all between calls?

What changes is only what we hand it — the parameter.

📖 New Vocabulary

Four words we need today

def

The keyword that starts a function definition.

parameter

A placeholder name inside the parentheses, e.g. name.

argument

The real value you hand over when calling, e.g. "Maya".

return

Sends a value back out of the function to be used later.

🤖 AI Moment

Same-answer function vs. an AI that improves

Our function today

Rule-based: given the same input, greet("Maya") ALWAYS prints the same greeting. It never changes unless we edit the code ourselves.

A learned AI system

Given more examples over time, it can change its own answer — it "learns" patterns instead of following one fixed rule.

🧠 Discussion question: Can you think of an app on your phone that behaves the same way every time, versus one that seems to "get smarter" as you use it?

💡 Deeper Look

A function that hands something back

def double(number):
    return number * 2

result = double(7)
print(result)  # 14

Notice:

print() just shows something on screen. return sends a value back so we can SAVE it in a variable and use it again later.

Which one would you need if you wanted to add the function's answer to another number?
