Function Composition — Chaining Machines Together
🔗 Concept — One Function's Output Becomes Another's Input

In today's lesson, greet(name) and double(number) each did one job. Composition means feeding the return value of one function directly into another.

def double(n):
    return n * 2

def add_five(n):
    return n + 5

# Composition: feed double's output straight into add_five
result = add_five(double(3))

🎛 Part 2 of 5 · 10 min
Higher-Order Functions — Passing a Function as an Argument
⚙️ Concept — A Function That Takes Another Function

So far, arguments have been numbers or strings — like greet("Maya"). A higher-order function takes a function itself as the argument, so you can swap the behavior without rewriting the outer function.

def shout(text):
    return text.upper() + "!!!"

def whisper(text):
    return text.lower() + "..."

def apply_style(name, style_function):
    message = style_function("ready to code")
    print(name + " says: " + message)

apply_style("Aiden", shout)
apply_style("Sofia", whisper)

Notice: apply_style never mentions upper() or lower() — it just trusts whatever function is handed to it. This is exactly the pattern behind Python's real sorted(list, key=...) and map().

🌀 Part 3 of 5 · 10 min
Recursion Lab — A Function That Calls Itself
🔁 Concept — The Base Case Stops the Spiral

Every recursive function needs two parts: a base case that stops the calls, and a recursive case that calls the function again with a smaller problem.

def countdown(n):
    if n <= 0:              # base case — stops the recursion
        print("Liftoff!")
        return
    print(n)
    countdown(n - 1)          # recursive case — smaller problem

countdown(3)

🐱 Part 4 of 5 · 8 min
Scratch Port Challenge — Same Logic, Different Language
🔄 Concept — Functions Exist in Scratch Too, as Custom Blocks

Scratch calls a function a custom block, built with Make a Block. It has the same two moves as Python: define the block once, then call (drag) it as many times as you like.

Python version (already built):

def countdown(n):
    if n <= 0:
        print("Liftoff!")
        return
    print(n)
    countdown(n - 1)

Scratch version — sketch the block stack:

define countdown (n) if <n ≤ 0> then say "Liftoff!" else say (n) call countdown (n − 1)

Your challenge: in Scratch, a custom block can call itself, just like Python. Describe (in words or a block sketch) how you would build sum_to(n) from Part 3 as a Scratch custom block.

🔍 Compare: what is the Scratch equivalent of Python's return keyword? (Hint: it's a reporter block, not a "say" block.)
