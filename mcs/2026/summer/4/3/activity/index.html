🔑 Key Ideas for Today · 今日重点
  • Function — a named block of code that runs only when you call it, like a recipe
  • def — the keyword that starts a function definition
  • Parameter — the placeholder name in the parentheses, like name
  • Argument — the real value you hand over, like "Maya"
  • return — sends a value back out so you can save and reuse it
💡 Predict before you run — if you can predict the output, you truly understand the code.

🤖 AI Moment — Same-Answer Function vs. an AI That Improves

Our greet() function is rule-based: the same input ALWAYS gives the same output. A learned AI system can change its answers as it sees more examples. Sort these four apps — write R (rule-based, same every time) or L (learns and changes):

A calculator app: 7 × 2 is always 14
A video app that recommends new videos the more you watch
Our greet("Maya") function from today
A photo app that gets better at recognizing your pet over time

Think: Why is it useful that a function is 100% predictable? When might you want an AI that changes instead?

🟢 Tier A — Call the Function (Buddy-Friendly)
📋 What to do

These two functions are already written for you. Type them exactly, then write your own CALLS with different arguments. You don't change the recipe — you just press the button!

def cheer(name): print(name + " is a coding star! 🌟") def countdown(): print("3... 2... 1... GO!")

Now write 3 calls of your own (use classmates' or family names):

✅ Tier A checklist
  • I typed both functions with def, the colon :, and the indent
  • I called cheer() at least twice with two different names
  • I called countdown() — did it need an argument?
  • I showed my buddy what printed

What was the SAME in every cheer() output, and what CHANGED?

🟡 Tier B — Refactor the Guessing Game
📋 What to do

Open your Week 3 number-guessing game. Find code you wrote more than once (hints, messages, checks) and wrap it inside a function you name yourself. The game must still work!

  • Find at least one repeated (or repeatable) chunk of code
  • Define a function with a clear name, like show_hint()
  • Call it from at least 2 places in your game

🟣 Tier C — Return a Value, Use It in a Calculation
📋 What to build

Write a function that returns a value, then USE that value in a calculation — for example, a function that returns a random secret number your game compares against.

  • Your function must use return — not just print
  • Save the result in a variable: secret = pick_secret(50)
  • Use the variable in math or a comparison (+, *, <, == ...)
  • Only the standard library — random is perfect

Explain: why does this need return instead of print?
