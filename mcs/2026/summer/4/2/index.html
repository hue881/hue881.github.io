Mini-Lesson — Part 1

From Scratch blocks to Python text

Remember Scratch?

You dragged if <> then and if <> then else blocks. The logic is identical — we're just typing it now.

IF touching edge? → THEN say "Ouch!"
ELSE → say "Cruising..."

Same logic, in Python

if age < 13:
  print("Kid zone!")
elif age < 18:
  print("Teen zone!")
else:
  print("Adult zone!")

Python checks conditions top to bottom and runs the first one that's true — then skips the rest. Indentation (the spaces) is what shows Python which lines belong to which branch.

Mini-Lesson — Part 3

Meet random — Python's dice roller

import random

secret = random.randint(1, 100)
# picks a whole number from 1 to 100, inclusive

Step 1

import random — always the first line. It loads Python's built-in randomness toolkit.

Step 2

random.randint(1, 100) — hands you back one random integer in that range, every time.

Now the computer has a secret number nobody — not even the code — "knows" in advance. That's the setup for our whole game.

AI Moment

Phase 2 · Sessions 8–14

"Higher / lower" is the simplest search strategy AI uses

Finding "50" in 1–100 — binary search

Guess 50 (middle)
Too lowToo high
Try 75
Try 25

Each guess cuts the space in half

1 → 100 numbers becomes 50, then 25, then 12... in just 7 guesses you can find any number up to 100.

Real AI uses this constantly

Search engines, databases, and even some machine-learning algorithms narrow down huge possibilities by repeatedly splitting them in half — exactly like your "higher/lower" strategy.

Question for later: If the number could be anywhere from 1 to 1,000,000 — would guessing "one at a time" or "cutting in half" find it faster? By how much?

Vocabulary · 6 Key Words

Words every Python coder knows

Conditional

Code that runs only when a condition is true — if / elif / else.

Boolean

A value that is only ever True or False — what every comparison produces.

Module

A toolkit of extra code you load with import, like random.

Indentation

The spaces before a line that tell Python which block it belongs to.

Loop

Code that repeats — our game uses while True: to keep asking for guesses.

Binary Search

Repeatedly cutting a search range in half — the strategy behind today's AI Moment.

Debug Corner · Common Bugs

Bugs are normal — let's squash them together

Bug 1: Used = instead of ==

if guess = secret: is a syntax error. A single = stores a value; double == compares two values.

Bug 2: Forgot int() on input

input() always returns text. Comparing text "50" to a number crashes — wrap it: int(input(...)).

Bug 3: Wrong indentation

If a line under if isn't indented, Python won't know it belongs there. Keep spacing consistent.

Bug 4: Loop never ends

Forgot break in the "correct guess" branch? The while True: loop runs forever. Add break to exit.

Pro move: read the error message's last line first — it usually names the exact problem and line number.
