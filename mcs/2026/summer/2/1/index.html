🎓 Mini-Lesson — Part 1

What is a Conditional?

In Real Life

IFit is raining THENtake an umbrella
IFI am hungry
THENeat a snack
ELSEkeep working
IFthe light is green THENwalk

In Scratch — what the blocks say

The if block — only runs when TRUE

IFtouching the edge?
THENsay "Oops, wall!"
↳ If NOT touching: nothing happens at all

The if … else block — always runs something

IFspace key is pressed?
THENsay "You pressed SPACE!"
ELSEsay "No key yet…"

A conditional lets a program make a decision. It checks a condition (true or false?) and runs different code depending on the answer — making programs interactive and smart.

🎓 Mini-Lesson — Part 2

if vs if-else — What's the difference?

🔵

The if block

Only acts when the condition is TRUE. If false — nothing happens.

Is it raining? 🌧️
YES ✅NO ❌
Take umbrella
(nothing)
🟢

The if-else block

Handles both cases — something happens whether true OR false.

Is it raining? 🌧️
YES ✅NO ❌
Take umbrella
Wear sunglasses

Key question: Does your program need to react only when something IS true, or does it need a different reaction when it is NOT true too?

Where do you find these blocks in Scratch?

🟧

Step 1

Click the orange "Control" category in the block palette on the left.

🔍

Step 2

Find the if <> then and if <> then … else blocks.

🔷

Step 3

Drag in a blue hexagon from "Sensing" or "Operators" to fill the diamond-shaped slot.

The hexagon slot holds your condition — it must be TRUE or FALSE

key [space] pressed? touching edge? touching [sprite]? mouse down? (score) > 10

Notice: The hexagon shape is a visual hint that only boolean (true/false) values fit there — just like puzzle pieces that must match!

"AI is a giant pile of if-then rules… sort of."

Is it a cat? — Decision Tree on the Board 🐱

Mystery Animal

Has whiskers?

YES ✅NO ❌
Says meow?
YESNO
CAT! 🐱
Maybe dog?
Not a cat 🐾

This IS a classifier!

Early AI systems were built exactly like this — huge trees of if-then rules, written by hand by programmers.

But modern AI is different…

Instead of humans writing every rule, modern AI learns the rules from thousands of examples. The rules become numbers (weights) — not code blocks!

Today's build is the first step toward understanding how classifiers work.

Question for later: What would happen if someone showed our "Is it a cat?" tree a lion? Has whiskers? ✅ Says meow? ❌ … Would our tree get it right? What does this tell us about AI?

📚 Vocabulary · 5 Key Words

Words every coder knows

🔀
Conditional

A code block that runs only when a certain condition is true. "IF this THEN that."

✅
Boolean

A value that is only ever TRUE or FALSE. The hexagon slot only accepts booleans!

🌿
Branch

One of the paths a program can take — the "THEN" path or the "ELSE" path.

🌳
Decision Tree

A diagram (or AI model) that uses a chain of conditions to classify or decide. Used in real AI!

🔁
Nested if

An if block placed inside another if block — checking a second condition only when the first is true.

🔍
Classifier

An AI system that sorts inputs into categories using learned rules — built on the same logic as decision trees.

🐛 Debug Corner · Common Bugs

Bugs are normal — let's find them together

Bug 1: Wrong block type

Dragging a round (reporter) block into the hexagon slot — they don't fit! Make sure you use a hexagon (boolean) block like key pressed?, not a value like x position.

Bug 2: if is outside the forever loop

If the if block is not inside a forever loop, it only checks the condition once — right when the flag is clicked. Most interactive projects need a continuous check!

Bug 3: Both say blocks show at once

You accidentally placed both say blocks in the same branch (both in "then" or both in "else"). Make sure one is in the THEN section and the other is in ELSE.

Bug 4: Nothing seems to happen

Did you click the green flag first? Is your sprite actually touching the edge or is the key the right one? Click on the condition block alone to test it — watch it flash yellow!

Pro move: Right-click any block and choose "Help" — Scratch will show you exactly what the block does.
