📖 Mini-Lesson · Big Idea #1

What Is a Variable?

A variable is a labeled container that stores one value.

You give it a name (like score), and it holds a number or text. You can read it, change it, or reset it at any time while the program runs.

Variable Name

score

Value Inside

0

Three things you can do with a variable:

1
Set it — give it a starting value
set score to 0
2
Change it — add or subtract
change score by 1
3
Read it — use it in a condition
if score = 10 then...
Think of it like a scoreboard: someone holds it up, changes the number when a point is scored, and shows it to everyone in the game.

📖 Mini-Lesson · Big Idea #2

Variables in Scratch — The Orange Blocks

Where to find them in Scratch:

Click the orange "Variables" category → "Make a Variable" → type a name → click OK.

The 4 key variable blocks:

set score to 0 → Reset at game start
change score by 1 → Add 1 point
show variable score → Display on stage
hide variable score → Make invisible

What does the orange oval do?

When you drag the orange score oval into another block, it reads the current value — like peeking inside the box.

Scope choice: Pick "For all sprites" so every sprite sees the same score. Pick "For this sprite only" for a private counter one sprite uses alone.

📖 Mini-Lesson · Big Idea #3

How Variables Look On-Screen

The variable badge on the Stage:

Scratch shows a little monitor on the Stage — drag it where you want it!

Three display modes — right-click to switch:

N
Normal read-out — shows name + value
e.g. "score: 7"
L
Large read-out — only the number, big!
e.g. "7" in large font
S
Slider — drag a handle to change value
Great for speed/difficulty variables!
Uncheck the checkbox next to the variable name in the palette to hide the badge from the stage — useful when you don't want to show the player a "lives" counter until game over.

AI Moment · Variables = Memory

Today's coding concept connects directly to how AI thinks — let's see how!

AI Keeps a "Score" Too — It's Called Confidence

Your Scratch game:

score = 0
// player catches a star
change score by 1
score = 1

Score starts at 0, grows with each good action.

Key insight: When the game ends, the score is forgotten — it resets when you click the green flag. AI models can be different — they save what they learned!

An AI model identifying a cat photo:

cat_confidence 87%
dog_confidence 13%
Both your score variable and the AI's confidence variable do the same job: store a changing number that the program checks to decide what to do next.

🔨 Guided Build · 20 min · Let's Build Together!

Mission: Add a Score to Your Scratch Game

We'll build a clicker/catch game where the score goes up every time the sprite is clicked or catches an object.

By the end of this build you'll have:

  • A variable named score
  • Score resets to 0 when game starts
  • Score goes up by 1 with each click
  • Score displayed on the stage

Open Scratch — let's walk through it step by step:

1
Go to Variables (orange category) → click "Make a Variable"
2
Name it score → choose "For all sprites" → click OK
3
In the sprite's script: drag when green flag clicked → set score to 0
4
Add a second script: when this sprite clicked → change score by 1
5
Press the green flag — click your sprite — watch the score grow!

🔨 Guided Build · The Two Scripts

Your Complete Code — Side by Side

Script 1 — Game Start

when green flag clicked set score to 0

This script runs once when the game starts. It makes sure the score always begins at zero — not at whatever number it was last time!

Script 2 — Scoring Points

when this sprite clicked change score by 1

This script runs every time the sprite is clicked. It adds 1 to whatever the current score is — and Scratch displays the new value instantly.

🟡

Why set to 0?

Without it, the score starts wherever it ended last game — try it to see what goes wrong!

🔢

Why "change by 1"?

"Change by" adds to the current value. "Set to 1" would just stay at 1 forever!

💡

What if score > 10?

Add an if block: if score = 10 → say "You Win!"

🔨 Guided Build · Level Up

Adding a Lives Counter — Two Variables at Once

Script 3 — Lives System

when green flag clicked set lives to 3 forever if touching [enemy] then change lives by -1 wait 1 seconds end

The wait 1 second is crucial — without it, lives drops to 0 in milliseconds!

Why negative? change lives by -1 subtracts 1. Think: changing by negative = going down.

Three tiers — same concept:

Tier A

Add score only. Make sure it resets and goes up by 1 when you click.

Tier B

Add both score (goes up) and lives (starts at 3, goes down on collision).

Tier C

Add a high-score variable — after the game, if score > highScore, update it!

Tier C challenge — High Score code:

// at end of game: if score > highScore then set highScore to score end

Quick bug check: Press green flag — does lives start at 3? Does score start at 0? If not, you're missing the set blocks!

🐛 Debugging · Common Variable Bugs

Bug Hunters — Can You Find the Fix?

Bug #1 — Score Starts at Old Number

// Missing reset! when green flag clicked // ← no "set score to 0" here

Fix: Always add set score to 0 under when green flag clicked.

Bug #2 — Score Never Changes

when this sprite clicked set score to 1 // ← "set to 1" stays at 1!

Fix: Change set score to 1 → change score by 1.

Bug #3 — Lives Vanishes Instantly

forever if touching [enemy] then change lives by -1 // ← no wait! loses life 60×/sec

Fix: Add wait 1 seconds after the change block.

Bug #4 — Score Invisible on Stage

// Variable monitor is unchecked! // OR: "hide variable" block ran

Fix: Check the checkbox next to score in the Variables palette, or add a show variable score block.

Debugging mindset: Every bug is your code being exact — it does exactly what you told it, even if that's not what you meant. Read your blocks out loud to find the mismatch!
