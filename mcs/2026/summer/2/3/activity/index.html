🔨 Part 3 · Guided Build (20 min)
Build Together — Step by Step

Follow along with your teacher. Check off each step as you complete it.

🛠 7-Step Build Checklist
Step 1 — Open Scratch & Pick Your Backdrop

Go to scratch.mit.edu → New Project → backdrop icon → choose your scene.

Step 2 — Choose Your Sprite

Delete the cat or keep it. Pick your player sprite. Resize to 60–80px.

Step 3 — Add the Green Flag Event

From Events, drag When 🏁 clicked onto the canvas. Everything starts here.

Why does the flag matter? Without it, your code runs never. The flag is your game’s “on” switch!
Step 4 — Make It Move

Inside a forever loop, add arrow-key conditionals. Test it now!

Arrow Keys: Key [right] pressed? inside If → Move 5 steps
Mouse: point toward mouse + Move 3 steps
Step 5 — Add a Second Sprite

Add something to catch, avoid, or collect. Use touching [sprite]? to detect collision.

Step 6 — Add a Score Variable

Variables → Make a Variable → name it “Score” → Set to 0 when flag clicked → Change by 1 when sprites touch.

If your variable is “score” in some places and “Score” in others, Scratch treats them as different variables! Be consistent.
Step 7 — Test & Fix the Big 3 Bugs
BugWhat you seeQuick fix
Code disconnectedNothing happens on flag clickSnap onto When 🏁 clicked
If outside foreverSprite ignores keys after first checkMove If INSIDE forever
Score not resettingScore keeps old value from last gameAdd Set Score to 0 after flag event
🏁 Before Open Build: Make sure your project starts on flag click, your sprite moves, and score changes on touch. That’s the minimum shippable product!
💃 Movement Break (5 min)
You Are the Sprite!
🗻 Human Sprite Activity

Stand up! Your teacher calls out instructions. Your body IS the sprite. Act out what Scratch would do.

  • “Green Flag” → freeze in start position
  • “Arrow Right” → step right 3 steps
  • “Arrow Left” → step left 3 steps
  • “Forever loop” → keep repeating the last action
  • “Touch edge” → bounce back and turn around
  • “Touch a classmate” → say “Got it!” and clap once (+1 point)
  • “Game Over” → sit back down slowly
  • “Reset” → return to starting position

    • After the activity: What did the “forever loop” feel like in your body?

    🛠 Part 5 · Open Build (30–40 min)
    Find Your Tier and Build!

    Find your tier below. Start there — then climb! Fast finishers: grab a Challenge Card or help a neighbor.

    Tier A — Foundations · Grades 1–3
    🆎 Goal: Working movement + 1 event reaction

    Use the starter from the guided build. Sprite moves with arrow keys and says something when it touches the edge.

    Your checklist:

  • When 🏁 clicked at the top
  • Sprite moves when I press arrow keys
  • Sprite says something when it touches edge
  • Click the flag → it starts!

    • Blocks to find & snap:

    When 🏁 clicked forever { } if key [right] pressed? move 5 steps if touching edge? say [Hello!] for 2 secs

    What does your sprite say when it touches the edge?

    Buddy tip: One person finds the block, the other snaps it on. Switch after each step!
    Tier B — Core · Grades 3–5
    ✅ Goal: A catch/chase game with score

    Player sprite, second sprite, score variable that goes up on touch, and a start message.

    Your checklist:

  • Player sprite moves smoothly
  • Second sprite appears/moves on its own
  • Score variable set to 0 at start
  • Score +1 when sprites touch
  • Start message appears
  • Green flag restarts everything cleanly

    • Code pattern:

    When 🏁 clicked
    → Set Score to 0
    → Say “Ready?” 1 sec
    → forever
    → if touching [sprite2]?
    → Change Score by 1

    What is your game called?

    How does a player WIN?

    Tier C — Accelerated · Grades 6–9
    🟣 Goal: Full game loop with difficulty scaling

    Add a Lives counter, a Game Over screen, and a speed-up mechanism so the game gets harder as score increases.

    Your checklist:

  • All Tier B goals complete
  • Lives variable (start 3, −1 on collision)
  • Game Over screen when lives = 0
  • Enemy speeds up every 5 points
  • Restart without refresh (broadcast “restart”)
  • Custom start screen with title

    • Think through:

    How do you make speed increase? (Hint: variable + change)

    What Scratch feature lets one sprite tell another to do something?

    Stretch: Add a sound effect with Sound blocks. Then try broadcasting a message from one sprite to another — this is how complex games coordinate multiple characters!
