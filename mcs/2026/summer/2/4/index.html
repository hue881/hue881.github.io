🎓 Mini-Lesson — Part 1

Scratch and Python: same brain, different voice

Scratch Block-based
when green flag clicked
say [Hello!]
repeat [5] do something 5 times
Python Text-based
# program starts here
print("Hello!")

for i in range(5):
    print("Hello!")

🔑 Key idea: In Scratch you snap blocks together. In Python you type the same idea as text. The logic — "do this 5 times" — is identical. Only the spelling changes!

🎓 Mini-Lesson — Part 2

print() — your first Python word

How it works

You type:

print("Good morning!")

Computer shows:

Good morning!
❓ What do you think happens if you type print() with nothing inside the parentheses?

Anatomy of a print statement

print("Hello, World!")

print — the function name (a built-in command)

( ) — parentheses: always needed to call a function

"Hello, World!" — a string: text wrapped in quotes

🧠 In Scratch you use a say block. In Python you call print(). Same job, different outfit!

🐛 Common mistake: Forgetting the quotes around your text. print(Hello) will error — Python thinks "Hello" is a variable name! Always use print("Hello").

🎓 Mini-Lesson — Part 3

The for loop: Scratch's "repeat" block, in text

In Scratch

repeat 5
say [Hello!]

The indented block runs 5 times

In Python

for i in range(5):
    print("Hello!")

The indented line runs 5 times — same idea!

The Anatomy

for … in — Python's loop keyword

i — a counter variable that changes each time

range(5) — "give me numbers 0,1,2,3,4" (5 total)

: + indent — the colon + 4-space indent is Python's way of saying "here's what goes inside"

❓ What do you think this prints?

for i in range(3):
    print("Coding is fun!")

Talk to your neighbor — predict it before you run it!

The Rules:

  1. Everyone stands in a circle

🐛 Debug Corner · Common Bugs

Bugs are how you know you're really coding

Bug 1 — Missing quotes

# Wrong:
print(Hello World)

# Right:
print("Hello World")

Text must always be wrapped in "double quotes"

Bug 2 — Missing indent in loop

# Wrong — no indent:
for i in range(3):
print("Hi")

# Right — 4 spaces:
for i in range(3):
    print("Hi")

Python needs exactly 4 spaces (or one Tab) to know what's inside the loop

Bug 3 — Forgetting the colon

# Wrong — no colon:
for i in range(3)

# Right — colon required:
for i in range(3):

The : at the end of for tells Python: "here comes the loop body"

Pro Move — Read the error message!

SyntaxError: invalid syntax (line 2)

Python error messages tell you the line number and the type of mistake. They're helpful clues, not mean messages!

📚 Vocabulary · 5 Key Words

Words you used today

function

A named command that does a specific job. print() is a built-in function. You call it by writing its name followed by parentheses.

string

Text data in Python — always wrapped in quotes. "Hello!" is a string. Numbers without quotes are numbers; with quotes they become strings.

loop

A block of code that repeats. A for loop runs a set number of times. Scratch's repeat is the same idea.

indentation

The spaces at the start of a line that tell Python what belongs inside a loop or function. In Scratch, the block snapping does this automatically — in Python, you add the spaces.

syntax

The spelling rules of a programming language. Python's syntax requires quotes around strings, colons after for, and indented loop bodies. Syntax errors = typos Python noticed.

comment

A note in your code that Python ignores (starts with #). Comments help humans understand the code — great for explaining what each section does!
