How Can a Robot Find Its Way Through a Maze?

Today students read a small grid, predict a robot’s choices, and watch how memory helps the robot avoid getting stuck while searching for a goal.

⏱️ About 45–60 Minutes 🤖 Grid + Robot Path 🐍 Python Thinking 🧠 Memory + Decision Making

Warm-Up

Do Now

Look first, think second, then explain your idea with a partner.

If a robot starts in a maze and always checks up, right, down, then left, what kind of path do you think it might make?

Write

  • What might the robot try first?
  • Could it miss the goal even if the goal is in the maze?
  • What problem might happen if the robot has no memory?

Turn and Talk

Compare answers with a partner. If your ideas differ, point to the maze and explain which square the robot would step on next and why.

Vocabulary

Words We Will Use

Grid
A layout made of rows and columns. Each box has a position.
Example: grid[2][3] means row 2, column 3.
Start
The place where the robot begins. In our maze, it is marked with S.
The search begins here.
Goal
The place the robot wants to reach. In our maze, it is marked with G.
The algorithm stops when the robot reaches it.
Visited
A memory of places the robot has already explored.
This stops the robot from repeating the same move forever.
Stack
A collection where the last item added is the first one taken out.
Like a stack of trays: you take the top one first.
Algorithm
A step-by-step rule for solving a problem.
Check directions in order, move, remember, repeat.

Maze Setup

Read the Map First

Before running code, students should be able to describe what the robot sees.

grid = [ list('S...#'), list('.#..#'), list('..#..'), list('#..G.') ] rows = len(grid) cols = len(grid[0]) for r in range(rows): print(''.join(grid[r]))

Notice

  • S is the start.
  • G is the goal.
  • # means wall.
  • . means open space.

Ask the Room

  • Which directions are blocked right away?
  • Which move choices seem safe?
  • How many possible paths can you already see?

Core Code

The Robot Gets Memory

Pause at each section and have students predict what the next line changes.

# Find start position S start = None for r in range(rows): for c in range(cols): if grid[r][c] == 'S': start = (r, c) break if start: break moves = [(-1, 0), (0, 1), (1, 0), (0, -1)] stack = [start] visited = {start} while stack: r, c = stack.pop() if grid[r][c] == 'G': print('Reached goal at', r, c) break if grid[r][c] == '.': grid[r][c] = '*' for dr, dc in moves: nr, nc = r + dr, c + dc if 0 <= nr < rows and 0 <= nc < cols: if grid[nr][nc] != '#' and (nr, nc) not in visited: visited.add((nr, nc)) stack.append((nr, nc))

Key thinking move

The robot is not choosing randomly. It checks positions in a fixed order, tests whether a move is safe, and stores new places so it can come back to them later.

Important pause points

  • Why is the start added to both stack and visited?
  • What would happen if (nr, nc) not in visited disappeared?
  • Why does stack.pop() take from the end?

Guided Practice

Trace the Search

1
Point to S. Ask students which direction the robot checks first. If that move is impossible, move to the next direction in the rule.
2
Mark each new open square with a pencil star or dot. Once a square is used, add it to the class memory list.
3
When students disagree about the next move, return to the code: check boundaries, check walls, then check memory.
4
After the robot reaches G, ask whether it found the shortest path or simply a path that works.

Notebook prompt

“The robot can avoid getting stuck because ______________________.”

Challenge prompt

“If the direction order changed, how might the path change?”

Big Idea

What Changes the Search?

QuestionWhat students should notice
No memoryThe robot may revisit the same spaces again and again, especially in a maze with loops.
With visited setThe robot remembers explored squares and avoids wasting moves.
Take from end with stack.pop()The newest choice is explored first, so the robot tends to go deep along one path before backing up.
Different direction orderThe robot may still reach the goal, but the path it draws can look very different.
Wall checks and boundary checksThese keep the robot safe from crashing into blocked spaces or leaving the grid.

Check for Understanding

Exit Ticket

Students answer in complete sentences using at least two vocabulary words.

Respond before you leave

1. Why does the robot need a visited set?

2. What does stack.pop() cause the robot to do?

3. If the robot checked left first instead of up first, what might change?
