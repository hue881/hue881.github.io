SLIDE 5 — JUST 4 WORDS TO REMEMBER
[Duration: ~3 minutes]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

"Now — our entire vocabulary for today. Four words. That's it. Let's go
through each one carefully."

"WORD 1: VOTE — A thumbs-up (1) or thumbs-down (0) from ONE person about
ONE thing.

  💡 Why do we use 1s and 0s? Because computers understand numbers, not
  feelings. When you say 'I like this game,' a computer needs to store that
  as a number. 1 means 'yes, I liked it,' 0 means 'no, I didn't.' This is
  called BINARY ENCODING — turning real-world information into numbers.
  Every piece of data your computer processes — photos, videos, songs,
  text — is ultimately stored as 1s and 0s. Today we're doing that at the
  simplest possible level."

"WORD 2: AVERAGE — Add all the votes, then divide by how many people voted.

  💡 The average is also called the ARITHMETIC MEAN in mathematics, and
  it represents the 'central tendency' of a group of numbers — basically,
  what's typical. In machine learning, averages are absolutely everywhere.
  When a neural network — the kind used in ChatGPT and image recognition —
  learns from data, it is constantly computing weighted averages. The
  humble average you learned in 3rd grade is the conceptual ancestor of
  the most powerful AI systems in the world."

"WORD 3: PICKER — The program that does the picking for us. Fancy term:
RECOMMENDER.

  💡 In professional computer science, this is called a 'recommendation
  engine' or 'recommender system.' It's an entire subfield of AI. There
  are academic conferences, research papers, and billion-dollar companies
  built around improving pickers. Amazon's recommendation system (the
  'customers also bought' section) reportedly generates 35% of their total
  revenue. That's the power of a good picker."

"WORD 4: TOP — The item with the highest average score. The one the picker
shows first.

  💡 In software, 'top' sounds simple, but choosing the right ranking
  criterion is one of the hardest problems in all of AI. What does 'top'
  even mean? Most liked? Most relevant? Most likely to keep the user
  engaged? These choices have enormous social consequences — what rises to
  'top' on social media shapes what millions of people think and believe.
  Today we rank by average score. That's our definition of 'top.'"

"Quick mental check — if 5 friends all thumbs-up a snack: [1,1,1,1,1],
what's the average?"
✋ [Students: 5/5 = 1.0]
"Right — perfect score, 1.0. What about [1,0,0,0,0]?"
✋ [Students: 1/5 = 0.2]
"Exactly. Now you have all the vocabulary you need to write real AI code."

SLIDE 6 — PENCIL & PAPER: FIND THE AVERAGE BY HAND
[Duration: ~5 minutes]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

"Before we touch code, we do it by HAND. This is important. When you
understand the math manually, you'll understand exactly what your code is
doing — and when something goes wrong, you'll be able to debug it."

"Let's walk through the worked example. Five friends voted on 'Racing':
  Votes: [1, 1, 0, 1, 1]

  Step 1 — Add the thumbs-ups: 1+1+0+1+1 = 4
  Step 2 — Count the voters: 5
  Step 3 — Divide: 4 ÷ 5 = 0.8

  80% thumbs-up score. Four out of five friends liked it."

💡 [CONNECT MATH TO CODE]:
"Here's what's beautiful: in Python, Step 1 is the built-in function
sum(), and Step 2 is the built-in function len(). So sum([1,1,0,1,1])
gives you 4, and len([1,1,0,1,1]) gives you 5. Python does the adding
and counting for you. Your job — your algorithm — is telling Python WHEN
to do those steps and WHAT to do with the result."

"Now you try the three puzzles. Pencils out."

  Puzzle:  [1, 0, 0, 1, 0] → answer: 2/5 = 0.4
  Shooter: [0, 0, 1, 0, 0] → answer: 1/5 = 0.2
  Sports:  [1, 1, 1, 1, 1] → answer: 5/5 = 1.0

✋ [Give 90 seconds, then go over answers. Have students hold up fingers to show their answer — 1 finger for Puzzle winning, 2 for Shooter, 3 for Sports]

"Sports wins with a perfect 1.0. Everyone loved it. That's what our
picker will figure out — automatically — when we write the code."

"Notice something: we just created what's called a RANKING. We put all
four options in order from best to worst: Sports (1.0), Racing (0.8),
Puzzle (0.4), Shooter (0.2). Ranking by average score is one of the
foundational operations in search engines, recommendation systems, and
basically all of modern AI."

SLIDE 7 — CODE STEP 1 OF 3: THE DICTIONARY
[Duration: ~4 minutes]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

[Have students open their Python editor / Replit / IDLE]

"Code time! We build in three steps — do NOT skip ahead. Type exactly

"Step 1: We store our data in a DICTIONARY."

[Walk around the room as students type. Dictate slowly:]

  ratings = {
    'Racing':  [1, 1, 0, 1, 1],
    'Puzzle':  [1, 0, 0, 1, 0],
    'Shooter': [0, 0, 1, 0, 0],
    'Sports':  [1, 1, 1, 1, 1],
  }
  print(ratings)

"Now run it."

✋ [Wait for students to run, help those with errors]

💡 [EXPLAIN THE DICTIONARY IN DEPTH — this is a critical concept]:
"Let's talk about what a dictionary IS. In Python, a dictionary is a
data structure that pairs a LABEL — called a KEY — with a piece of DATA
— called a VALUE. Think of it like a physical dictionary: you look up a
WORD (the key) and find its DEFINITION (the value).

In our code:
  - 'Racing' is a KEY — the label for one game
  - [1, 1, 0, 1, 1] is the VALUE — the votes for that game

Why use a dictionary instead of separate variables? Because now we can
write code that works on ALL games at once, without knowing in advance
how many games there are. That's scalability — the ability to handle
bigger and bigger datasets with the same code. Netflix has 17,000 titles.
They don't write a separate line for each one. They use data structures
exactly like this dictionary."

"The key-value pair structure of a dictionary shows up EVERYWHERE in
computer science:
  - A phone contact list: name → phone number
  - A login system: username → password hash
  - A shopping cart: item name → price
  - A word counter: word → count

It is one of the most important data structures you will ever use."

SLIDE 8 — CODE STEP 2 OF 3: THE LOOP & AVERAGE
[Duration: ~5 minutes]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

"Step 2: We loop through all the games and compute the average score for
each one. Type this BELOW your Step 1 code:"

  averages = {}
  for game, votes in ratings.items():
      score = sum(votes) / len(votes)
      averages[game] = score

  for game, score in averages.items():
      print(game, '->', round(score, 2))

"Run it. Did Sports come out as 1.0?"

✋ [Wait — help students troubleshoot indentation errors, which are the most common issue here]

💡 [LINE BY LINE EXPLANATION — go slowly, this is the heart of the lesson]:

"Let's read this code like a story. Every line has a job:

  LINE 1: averages = {}
  → We create a new, EMPTY dictionary called 'averages.' Think of it as
    an empty scoreboard. No scores yet — we're about to fill it in.

  LINE 2: for game, votes in ratings.items():
  → This is a FOR LOOP. In Python, a for loop says: 'Go through every
    item in this collection, one at a time, and do something with each.'

    ratings.items() gives us each key-value pair. So on the FIRST pass,
    game = 'Racing' and votes = [1, 1, 0, 1, 1]. On the SECOND pass,
    game = 'Puzzle' and votes = [1, 0, 0, 1, 0]. And so on.

    💡 DEEP INSIGHT: This is called ITERATION — visiting each element in
    a collection exactly once. Iteration is the mechanism behind almost
    every algorithm in computer science. When Google indexes the web,
    it iterates over billions of pages. When a face recognition system
    scans a photo, it iterates over millions of pixels. The FOR loop is
    the basic unit of that process.

  LINE 3: score = sum(votes) / len(votes)
  → This is our average formula — the same math you did by hand.
    sum(votes) adds all the 1s and 0s. len(votes) counts how many voters
    there were. Divide: you get the score.

    💡 In machine learning, this exact operation has a formal name:
    it's computing a MEAN RATING. The entire field of 'collaborative
    filtering' — used by Amazon, Netflix, and Spotify — extends this
    simple mean to consider patterns across thousands of users and items.
    The foundation is this one line of code.

  LINE 4: averages[game] = score
  → We write the score onto our scoreboard. We're saying: 'In the
    averages dictionary, the entry for THIS game equals THIS score.'
    After the loop finishes, averages contains a score for every game.

  LINES 5-6: The print loop
  → We loop through averages and print each game's score, rounded to
    2 decimal places. round(..., 2) is just cosmetic — it keeps our
    output clean and readable. In a real app, this would be the data
    sent to a display or a database."

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

💡 [DEEP EXPLANATION OF max() WITH A KEY FUNCTION — this is subtle]:

"Let's break down that one line:

  max(averages, key=averages.get)

  max() normally finds the biggest NUMBER in a list. But averages is a
  DICTIONARY — it contains key-value pairs, not just numbers. So we need
  to tell max() a rule: 'When comparing items, compare them BY their
  score, not by their name.'

  The argument key=averages.get does exactly that. averages.get is a
  function that, given a game name, returns its score. So we're saying:
  'Find the game whose score, as reported by averages.get, is the maximum.'

  💡 This pattern — max(collection, key=some_function) — is one of the
  most powerful one-liners in Python. In professional code you'll see it
  everywhere. When ranking search results, finding the fastest route,
  picking the highest-paying job offer, selecting the most statistically
  significant result in an experiment — all of these use this same pattern.

  The technical term for what we just built is an ARGMAX operation:
  'Find the ARGUMENT (input) that produces the MAXIMUM output.' It's a
  fundamental operation in calculus, optimization theory, machine learning
  training, and game theory. You just wrote argmax. In 8 lines."

"Let's zoom out. We have:
  1. Stored structured data → dictionary
  2. Processed it with a loop → iteration
  3. Applied a mathematical operation → mean
  4. Selected the best result → argmax

That is not a toy exercise. That is a complete data pipeline. The only
difference between this and Netflix's system is scale: they have 200
million users, you have 5. The logic is identical."

SLIDE 11 — TALK TIME: WHEN DOES "MOST POPULAR" GO WRONG?
[Duration: ~4 minutes]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

"Our picker is smart — but it has real problems. Every algorithm does.
Let's be critical thinkers. There are three specific failure modes on the

"PROBLEM 1: Tastes are different.
If 19 kids love chocolate but YOU love mint, the popular pick is wrong —
for you. In AI, this is called the MAJORITY BIAS or POPULARITY BIAS. The
algorithm unfairly favors items that already have lots of votes, which
tends to be things that mainstream, majority groups prefer. For minority
tastes, popular-based recommendations fail."

"PROBLEM 2: Brand new things get zero votes.
A new game or song starts with ZERO data. Our algorithm can't possibly
recommend it — it has no score. In the tech world this is called the
COLD START PROBLEM. It's one of the hardest problems in recommendation
systems. How do you recommend something you've never seen data on? This
is why apps often ask new users to rate a bunch of things immediately
after signing up — they need data to warm the system up."

"PROBLEM 3: You only see what you already like.
If the algorithm always recommends things similar to what's already
popular with you, you end up in what researchers call a FILTER BUBBLE
or ECHO CHAMBER. You only see one type of content, one type of opinion,
one type of music — forever. This has real social consequences. It's one
reason why social media can feel so divided — everyone is in their own
recommendation bubble."

technical bugs. They're ETHICAL questions. Who should decide what 'top'
means? Who is responsible when an algorithm amplifies harmful content?

users — the consequences are enormous. This is why AI Ethics is now
a major field in computer science. The code you wrote in 8 lines raises
real questions that the biggest tech companies in the world are still

  player_ratings = {
    'Alex':   {'Racing':[1,1,1], 'Puzzle':[1,1,0], 'Shooter':[0,0,0]},
    'Jordan': {'Racing':[0,1,0], 'Puzzle':[1,1,1], 'Shooter':[1,1,0]},
  }

  for name, games in player_ratings.items():
      print('
For', name)
      for game, votes in games.items():
          score = sum(votes) / len(votes)
          print(' ', game, '->', round(score, 2))

"Before you run it — PREDICT: who likes Racing more, Alex or Jordan?"
✋ [Let students guess — Alex has [1,1,1]=1.0 vs Jordan's [0,1,0]=0.33]

"Run it. Does each person get their own list?"

💡 [DEEP EXPLANATION — introduce collaborative filtering]:
"Look at the structure of player_ratings. It's a dictionary of
dictionaries — or in computer science terms, a NESTED DATA STRUCTURE.
The outer dictionary maps person names to their data. The inner
dictionary maps game names to votes. Two levels of organization.

This data structure, if you scaled it up, becomes what's called a
USER-ITEM MATRIX — the foundational data structure of collaborative
filtering. Imagine a giant table:

              Racing  Puzzle  Shooter  Sports
  Alex:         1.0    0.67     0.0     ???
  Jordan:       0.33   1.0      0.67    ???

Each row is a USER. Each column is an ITEM. The values are average
ratings. Notice the question marks — Alex hasn't rated Sports yet.
How do we fill in that blank?

That's exactly what Netflix's recommendation engine does. It looks at
your row, finds users whose rows look SIMILAR to yours, and predicts
what YOU would rate the items you haven't seen yet — based on what
SIMILAR users rated. This technique is called USER-BASED COLLABORATIVE
FILTERING, and it was the winning algorithm in the famous Netflix Prize —
a $1 million competition in 2009 that transformed the field of AI.

What you just coded — two levels of nested loops, two levels of nested
dictionaries — is the SKELETON of that $1 million algorithm."

APPENDIX A — COMMON ERRORS & HOW TO FIX THEM
================================================================================

ERROR: IndentationError
  Cause: Python uses indentation to define code blocks. One extra or
         missing space breaks everything.
  Fix:   Make sure every line inside a `for` loop is indented exactly
         4 spaces (or one tab consistently). Use the TAB key, not spacebar.

ERROR: KeyError or TypeError
  Cause: Mistyped a game name, or tried to use a string where a number
         was expected.
  Fix:   Check that your dictionary keys match EXACTLY — including
         capitalization and quotes.

ERROR: ZeroDivisionError
  Cause: An empty vote list [] causes len(votes) = 0, and division by 0
         crashes Python.
  Fix:   Make sure every game has at least one vote in its list.

ERROR: Output is wrong / not what expected
  Cause: Data entry error — a 1 typed where a 0 should be, etc.
  Fix:   Manually compute the average on paper, then check whether the
         code matches. This is debugging by verification.
