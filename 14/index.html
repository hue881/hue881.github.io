Vocabulary · Use These Words Today

The 5 key words 🔑

⚖️ Weight (w)

How much the neuron cares about an input. A higher weight means that input matters more to the final answer.

📍 Bias (b)

A nudge that shifts the neuron's answer up or down even when inputs are zero. It's like a head-start or handicap.

🎯 Target

The correct answer we want the neuron to produce — 1 for "big", 0 for "small". Ground truth.

❌ Error

target − prediction. If positive, we guessed too low. If negative, too high. Zero = perfect prediction.

🐢 Learning Rate (lr)

How big each update step is. Too large → overshoots; too small → trains slowly. Think of it as the "step size".

Quick Check

If w = 0.1 and we want the neuron to pay more attention to x, should w go up or down?

Core Concept · The Neuron Formula

What does a single neuron actually do? ⚡

The forward pass (1 input)

raw = w × x + b pred = 1 if raw >= 0.5 else 0

The update rule

error = target - pred w = w + lr × error × x b = b + lr × error

?Think it through

  • If w=0.1, b=0, x=3 — what is raw? Is pred 0 or 1?
  • Now try x=15 — does pred change? Why?
  • If the neuron predicts 0 but the target is 1, is error positive or negative?
  • When error is positive, which direction does w move — up or down? Does that make sense?

🧩 Big idea: The neuron doesn't "know" anything — it just follows these two formulas. But after 40 steps, it gets surprisingly good. Why?

Activity 1 · One-Neuron Number Classifier

Can one formula learn "big" vs "small"? 🔢

Python — run this in your notebook

import random w = 0.1 # weight for x b = 0.0 # bias lr = 0.05 # learning rate for step in range(40): x = random.randint(0, 20) target = 1 if x >= 10 else 0 # Forward pass raw = w * x + b pred = 1 if raw >= 0.5 else 0 error = target - pred # Update w = w + lr * error * x b = b + lr * error print('step', step, 'x', x, 'target', target, 'pred', pred, 'w', round(w,2), 'b', round(b,2))

  • Count mistakes in steps 0–9 vs steps 30–39. What do you notice?
  • What happens to w over time — does it keep growing forever? Why does it stop?

Why do weights and biases exist? 🎚️

🎸The guitar analogy

  • Weights are like a volume knob for each input — they amplify or reduce how much it matters.
  • Bias is like the guitar tuner peg — it shifts everything up or down even before any input arrives.
  • During training, the network compares predictions to answers and tweaks both to reduce error.

Activity 2 · Two Inputs: Homework + Quiz

Now the neuron judges two scores 📋

Python — two-input version

import random w1, w2 = 0.1, 0.1 b = 0.0 lr = 0.05 for step in range(40): x1 = random.randint(0, 10) # homework x2 = random.randint(0, 10) # quiz total = x1 + x2 target = 1 if total >= 12 else 0 raw = w1*x1 + w2*x2 + b pred = 1 if raw >= 0.5 else 0 error = target - pred w1 = w1 + lr * error * x1 w2 = w2 + lr * error * x2 b = b + lr * error print('x1',x1,'x2',x2,'target',target,'pred',pred, 'w1',round(w1,2),'w2',round(w2,2),'b',round(b,2))

?Deeper investigation

  • Will the neuron treat homework (x1) and quizzes (x2) equally? How would you know from w1 and w2?
  • If w1 ends up larger than w2, what does that say about what the neuron "cares" about more?
  • Could you set up the scores so x1 always matters more? What would the data need to look like?
  • Is this fair grading? If a student aces homework but bombs quizzes — what does the neuron predict?

🏁 Extend: Change threshold from 12 to 8. Does w1 ≈ w2 still hold after training?

Deep Dive · The Training Loop

How learning actually happens — step by step 🔄

The learning loop — every single step

① Get a random input x

A new number 0–20 each step — the neuron has never seen it before

② Compute raw = w×x + b

Forward pass: apply current weights to get a raw score

③ Threshold → pred (0 or 1)

Make a decision: is this number big or small?

④ Compare to target → error

Were we right? error = target − pred

⑤ Update w and b

Nudge weights toward better performance — then repeat!

?The big questions

  • The loop runs 40 times. Is 40 a lot? Could you learn to ride a bike in 40 tries?
  • Why does the neuron need random inputs? What would happen if x was always 5?
  • Step ⑤ only runs when there's an error. What if error = 0 — does w change? Should it?
  • After 40 steps, the weights "settle". What would un-settle them?

🔗 Real world: This exact loop — see data → make prediction → measure error → update — is the core of every AI system, from spam filters to AlphaGo to image recognition on your phone.

Class Discussion · The Big Question

Is the neuron "thinking" — or just adjusting numbers? 🤔

Two sides of the debate

Side A: It IS thinking

  • It gets better at a task without being told how
  • It adjusts based on its own mistakes
  • With enough neurons it can recognize faces, translate languages, write poetry

Side B: It is NOT thinking

  • It follows a fixed rule — w = w + lr × error × x — every time
  • It has no understanding of "big" or "small" as concepts
  • It can't explain its own answer or wonder "why?"

Video Extension · The Update Rule Unpacked

Why does the update formula work? 📐

Watch · Perceptron Learning Rule

Source: YouTube · L3.2 The Perceptron Learning Rule — Sebastian Raschka

📊Math in plain English

If pred = target → no update

error = 0 → w + 0.05 × 0 × x = w. Weights stay put — we were right!

If we predicted 0, target = 1 → error = +1

w increases: neuron becomes more sensitive to x next time

If we predicted 1, target = 0 → error = −1

w decreases: neuron becomes less sensitive — it overreacted

⚡ If the data is linearly separable (a line can split 0s from 1s), the perceptron is guaranteed to find the answer in finite steps.
