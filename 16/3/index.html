📸 Core Concept · Images as Numbers

Every picture is secretly a grid of numbers 🔢

What is a pixel?

  • A pixel is the tiniest dot in a picture — like one square on graph paper.
  • Each pixel stores a number: 0 = pitch black, 255 = pure white.
  • A 1000×1000 photo contains 1,000,000 numbers — one per pixel!
  • A color photo has 3 grids (Red, Green, Blue) layered on top of each other.

🔍 Core Concept · What Is an Edge?

An edge is where numbers change fast ⚡

🌄

In real photos

An edge is where the brightness jumps suddenly — like where a dark cat meets a bright wall. Your eye sees a line. The computer sees a big number change.

🔢

In our number grid

Going from column to column:
0 → 0 → 5 → 5 → 0
The jump from 0 to 5 = an edge! The jump from 5 back to 0 = another edge!

🧠

Why AI needs edges

Edges define shapes. Shapes define objects. If AI can find edges, it can recognize letters, faces, stop signs, and tumors — even before knowing any colors.

🧮 Core Concept · The Filter Formula

One formula, one pixel at a time ⚙️

The Laplacian-style formula

value = center×4 − (up + down + left + right)

What it measures: how different the center pixel is from its 4 neighbors. If the center and its neighbors are all similar → the answer is close to zero (flat, boring region). If the center is very different → the answer is large (edge!)

📐 This is called the Laplacian operator — invented by mathematician Pierre-Simon Laplace in the 1700s. He had no idea it would one day help phones unlock with your face!

🔑 Vocabulary · Use These Words Today

The key words of image AI 📖

🖼️ Pixel

The smallest unit in a digital image. Each pixel stores one brightness number (0=black, 255=white). Millions of pixels together make one photo.

🔢 Grayscale Grid

A 2D list of numbers representing an image with no color — only brightness values. Perfect for learning the math before adding color.

📐 Edge

A location in the image where brightness changes rapidly between neighboring pixels. Edges outline shapes, objects, and boundaries.

🌀 Filter / Kernel

A small math operation slid over every pixel, combining it with its neighbors. Different kernels = blur, sharpen, or edge-detect.

🔁 Nested Loop

A loop inside another loop. The outer loop moves through rows (y), the inner loop through columns (x) — visiting every pixel one-by-one.

🐍 The Code · 5×5 Image Filter

Reading the code, line by line 🔬

image = [
  [0, 0, 0, 0, 0],
  [0, 5, 5, 5, 0],
  [0, 5, 9, 5, 0],   # brightest!
  [0, 5, 5, 5, 0],
  [0, 0, 0, 0, 0],
]

filtered = []
for y in range(1, 4):   # rows 1,2,3
  new_row = []
  for x in range(1, 4): # cols 1,2,3
    center = image[y][x]
    up     = image[y-1][x]
    down   = image[y+1][x]
    left   = image[y][x-1]
    right  = image[y][x+1]

    value = center*4 - (up + down + left + right)
    new_row.append(value)
  filtered.append(new_row)

Code Questions

Why does range(1, 4) skip row 0 and row 4? What is at the edges of the grid?

What does image[y-1][x] mean? Which pixel is that — above, below, left, or right?

If center = 5 and all 4 neighbors = 5, what will value be? Does that make sense?

If center = 9 and all 4 neighbors = 5, calculate value by hand. What does a big positive answer mean?

🔁 Deep Dive · Nested Loops

A loop inside a loop — visiting every pixel 🗺️

How the loops move

[1][1]
[1][2]
[1][3]
[2][1]
[2][2]
[2][3]
[3][1]
[3][2]
[3][3]

🔵 Row 1, 🟢 Row 2, 🟡 Row 3 — the outer loop picks the row, the inner loop walks across each column. Together they visit all 9 inner pixels.

Why only rows 1–3 (not 0 or 4)?

The formula needs an up, down, left, and right neighbor for every pixel. Row 0 has no row above it. Row 4 has no row below. Same for columns 0 and 4 — they sit on the edge of the grid itself, so we skip them.

💡 This is called a boundary condition — a rule to avoid reading outside our list. Real image filters use padding or mirroring to handle boundaries on larger images.

🧠 Going Deeper · Convolution

What is convolution — and why is it everywhere? 🌐

The big picture

Step 1 — Define a kernel

A tiny matrix of weights (like our ×4 and −1 values)

Step 2 — Slide it over the image

Visit every inner pixel with a nested loop (exactly what we coded!)

Step 3 — Compute weighted sum

Multiply pixel values by the kernel weights, add them up

Step 4 — Store output

New grid = the "filtered" image. Apply threshold to find edges.

Convolution is the engine inside Convolutional Neural Networks (CNNs) — the AI architecture behind image recognition. Instead of hand-coding one kernel, CNNs learn thousands of kernels automatically by training on millions of photos.

🔗 Your code is layer 0 of deep learning. Real CNNs stack 100+ convolution layers, each finding more complex features — from edges → shapes → objects → scenes.
