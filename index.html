🔑 Key Vocabulary

7 Terms You Need Today

Training Data

Examples the model learns patterns from; quality determines output quality

Bias

Systematic error that unfairly skews predictions toward one group

Class Imbalance

When one category has far more examples (e.g., 900 cats vs 10 dogs)

Tuple

('Alex', 'red') — immutable pair; ideal for records

Counter

Dict subclass: Counter(['a','b','a']) → {'a':2,'b':1}

List Comprehension

[x*2 for x in nums if x>0] — 1-line filter + transform

Fairness Metric

A number that measures how equally a model treats different groups

🐍 Python · Level Up

The Biased Robot — Annotated

bias_demo.py 
from collections import Counter

# Each tuple = (name, label)  ← immutable record
players = [
    ('Alex',  'red'),
    ('Blake', 'blue'),
    ('Casey', 'blue'),
    ('Devon', 'blue'),
    ('Emi',   'red'),
]

# Counter tallies every label in ONE line
counts = Counter(team for _, team in players)
# counts → Counter({'blue': 3, 'red': 2})

# Imbalance ratio — how skewed is the data?
total   = sum(counts.values())
ratio   = counts['blue'] / counts['red']
print(f"Ratio blue:red = {ratio:.2f}x")

# Most common label — what the robot will prefer
dominant = counts.most_common(1)[0]
print(f"Robot prefers: {dominant[0]} ({dominant[1]}/{total})")

Output 📤

Ratio blue:red = 1.50x
Robot prefers: blue (3/5)

New Concepts 🔬

  • Counter — dict that counts automatically
  • most_common(n) — top-n by frequency
  • f-string — embed values in strings
  • {:.2f} — round to 2 decimal places

🐍 Python · Iteration Patterns

Unpacking, enumerate & zip

patterns.py 
# ── Pattern 1: tuple unpacking ──────
for name, team in players:
    print(name, "→", team)

# ── Pattern 2: enumerate (index + value)
for idx, (name, team) in enumerate(players):
    print(f"#{idx}: {name} is {team}")

# ── Pattern 3: list comprehension ───
reds = [n for n, t in players if t == 'red']
# → ['Alex', 'Emi']

# ── Pattern 4: zip two lists ────────
names  = ['Alex', 'Blake', 'Casey']
scores = [0.9, 0.7, 0.8]
for name, score in zip(names, scores):
    print(f"{name}: {score:.0%}")

Pattern 1: Unpack

Python assigns each tuple element to its own variable — no indexing needed. name, team = ('Alex', 'red')

Pattern 2: enumerate

Gives you both the position and the value — better than writing i = 0; i += 1 by hand.

Pattern 3: Comprehension

Filter + transform in one line. Pythonic and fast. [expr for var in iterable if cond]

Pattern 4: zip

Pairs two lists element-by-element. If lists are different lengths, stops at the shorter one.

Why Counter beats a for-loop

Counter handles the dict-or-add logic automatically. It also handles new keys (no KeyError). Built-in speed: written in C, ~4× faster than manual counting on large datasets.

🌍 Real-World Impact

Three AI Failures — With Numbers

🏢

Amazon Hiring AI

2014–2018 · Decommissioned

Trained on 10 years of résumés that were 96% male. Learned to penalise CVs containing the word "women's" (e.g., women's chess club). Downgraded graduates of 2 all-women's colleges by default.

96% male training data → model reproduced the bias

👁️

MIT Face Recognition

Joy Buolamwini, 2018

Three major commercial face-recognition systems tested across skin tones and genders. Error rates ranged from 0.8% for light-skinned men to 34.7% for dark-skinned women — a 43× gap.

34.7% error vs 0.8% error = 43× disparity

🏥

NHS Medical AI

UK · 2021 Audit

Chest X-ray AI trained mostly on data from patients of one ethnicity. When used across all NHS patients, accuracy dropped 11% for under-represented groups. Affected triage decisions for ~140,000 scans/year.

11% accuracy drop on 140K+ scans per year

🛠️ Three Fix Strategies

Balancing the Dataset — Three Ways

Strategy 1 · Collect More

more = [('Finley', 'red'),
        ('Gray',   'red')]
data = players + more

counts = Counter(
    t for _, t in data
)
# Counter({'red':4,'blue':3})

✅ Best when possible. More real data is always better than synthetic data. Gap: 4 vs 3 (ratio 1.33×).

Strategy 2 · Undersample

import random

blues = [p for p in players
         if p[1] == 'blue']
reds  = [p for p in players
         if p[1] == 'red']

n = min(len(blues), len(reds))
balanced = (random.sample(blues, n)
          + random.sample(reds,  n))

⚠️ Loses data. Pick only as many of the majority as the minority has. Quick but wasteful.

Strategy 3 · Weight by Class

counts = Counter(
    t for _, t in players
)
total  = sum(counts.values())

# Weight = inverse of frequency
weights = {
    label: total / count
    for label, count
    in counts.items()
}
# {'blue':1.67, 'red':2.5}

✅ No data lost. Tell the model to care more about rare examples. Dict comprehension in action.

🐍 Python · Dictionaries

Dict Comprehension & .items()

dict_patterns.py 
from collections import Counter

players = [('Alex','red'), ('Blake','blue'),
           ('Casey','blue'), ('Devon','blue'),
           ('Emi','red')]

counts = Counter(t for _, t in players)
total  = sum(counts.values())

# ── Dict comprehension ─────────────────
# {key: value  for key, value in iterable}
pct = {
    label: round(count / total * 100, 1)
    for label, count in counts.items()
}
# pct → {'blue': 60.0, 'red': 40.0}

# ── Iterate the result dict ─────────────
for label, percent in pct.items():
    bar = "█" * int(percent // 10)
    print(f"{label:5s} {bar} {percent}%")

Output 📤

blue ██████ 60.0%
red ████ 40.0%

Key Methods

  • .items() → (key, value) pairs
  • .keys() → just the keys
  • .values() → just the values
  • round(x, n) → n decimal places
  • f"{x:5s}" → pad string to 5 chars

⚖️ Fairness Metrics

What Does "Fair" Actually Mean?

🎯

Demographic Parity

The model gives positive outcomes to equal proportions of each group, regardless of actual merit.

P(ŷ=1|Blue) ≈ P(ŷ=1|Red)

⚠️ Can force unfair results if groups genuinely differ.

✅

Equal Opportunity

Among people who should be selected (true positives), the model picks equal fractions from each group.

TPR(Blue) ≈ TPR(Red)

✅ Most commonly used in hiring AI and medical triage.

📐

Calibration

When the model says "70% confident," it should be correct 70% of the time — for every group equally.

P(y=1|ŷ=0.7) ≈ 0.7 ∀ groups

📊 Critical for medical AI and risk scoring tools.

🧠 Impossibility theorem (Chouldechova, 2017): You cannot satisfy all three fairness definitions simultaneously when base rates differ between groups. Every AI system must make a choice.

Build the Fairness Checker

your_challenge.py ← complete this! 
from collections import Counter

# Dataset: (name, team, model_prediction)
results = [
    ('Alex',  'red',  'red'),   # correct ✓
    ('Blake', 'blue', 'blue'),  # correct ✓
    ('Casey', 'blue', 'blue'),  # correct ✓
    ('Devon', 'blue', 'red'),   # WRONG  ✗
    ('Emi',   'red',  'blue'),  # WRONG  ✗
    ('Finley','red',  'red'),   # correct ✓
]

# ── Your task: ──────────────────────────
# 1) Count correct predictions per team
# 2) Count total examples per team
# 3) Print accuracy for each team
# 4) Flag teams where accuracy < 70%
# Hint: use Counter + dict comprehension

Expected Output 📤

blue: 2/3 = 66.7% ⚠️ BIAS
red: 2/3 = 66.7% ⚠️ BIAS

💡 Click to reveal one approach →

correct = {t: sum(1 for _,team,pred
             in results
             if team==t and pred==t)
           for t in ['red','blue']}
total   = Counter(t for _,t,_ in results)
for t in total:
  acc = correct[t]/total[t]*100
  flag = ' ⚠️ BIAS' if acc<70 else ''
  print(f"{t}: {correct[t]}/{total[t]}"
        f" = {acc:.1f}%{flag}")

👥 Ethics · Responsibility

Humans in the Loop — At Every Stage

🧑‍💻

Data Collectors

Decide what to record and what to leave out. Every gap is a choice with consequences.

Responsibility: completeness & representativeness

🏷️

Labellers

Apply tags that the model will learn from. Their biases become the model's biases.

Responsibility: consistency & clarity

⚙️

Model Builders

Choose the algorithm, loss function, and fairness constraints. No choice is neutral.

Responsibility: fairness metric selection

🏛️

Deployers & Auditors

Monitor the model in the real world. Catch drift, abuse, and unforeseen harm.

Responsibility: ongoing monitoring

🧠 The EU AI Act (2024) requires high-risk AI systems (hiring, medical, credit scoring) to have a human review every automated decision that significantly affects a person. Estimated to cover 6,000+ systems in deployment across Europe.
