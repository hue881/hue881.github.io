⭐ Project 1 · RateRec  🟢 BEGINNER

⭐ RateRec — Average-Rating & Per-Player Recommender

🟢 Beginner · Pure Python · No imports needed · ~15 lines total

recommender_v2.py 
# Per-player recommender (Day 15 upgrade)
player_ratings = {
    'Alex':   {'Racing':[1,1,1],
                'Puzzle':[1,1,0],
                'Shooter':[0,0,0]},
    'Jordan': {'Racing':[0,1,0],
                'Puzzle':[1,1,1],
                'Shooter':[1,1,0]},
}

for name, games in player_ratings.items():
    print('\nRecommendations for', name)
    averages = {}
    for game, votes in games.items():
        averages[game] = sum(votes) / len(votes)
    ranked = sorted(averages.items(),
        key=lambda item: item[1],
        reverse=True)
    for game, score in ranked:
        print(' ', game, '->', round(score, 2))

🔑 Key Concepts

  • .items() yields (key, value) pairs — outer loop = players, inner = games
  • lambda item: item[1] tells sorted() to sort by the score, not the name
  • reverse=True puts the highest score first — that's your recommendation

🔍 Project 2 · EdgeBot  🟡 INTERMEDIATE

🔍 EdgeBot — The 5×5 Laplacian Image Filter

🟡 Intermediate · Nested loops · 2-D list indexing · boundary math

edge_filter.py 
# EdgeBot — Laplacian edge detection (Day 16)
image = [
    [0, 0, 0, 0, 0],
    [0, 5, 5, 5, 0],
    [0, 5, 9, 5, 0],  # ← brightest pixel
    [0, 5, 5, 5, 0],
    [0, 0, 0, 0, 0],
]

filtered = []
for y in range(1, 4):    # skip edge rows
    new_row = []
    for x in range(1, 4):  # skip edge cols
        center = image[y][x]
        up     = image[y-1][x]
        down   = image[y+1][x]
        left   = image[y][x-1]
        right  = image[y][x+1]
        value  = center*4 - (up + down + left + right)
        new_row.append(value)
    filtered.append(new_row)

print(filtered)

🔑 Key Concepts

  • Boundary condition: range(1,4) skips row/col 0 and 4 because they have no outer neighbor
  • The formula center×4 − neighbors measures how different a pixel is from its surroundings
  • Big output value → rapid brightness change → edge detected!

💡 Upgrade Hook

Change image[2][2] = 9 to 1. Predict the new filtered values before running. Then add a second loop that prints "EDGE!" for any value > 8.

🤔 Think About It

If center = 5 and all neighbors = 5, the formula gives 0. If center = 9 and all neighbors = 5, calculate 9×4 − (5+5+5+5) by hand. What does that number mean?

🤖 Project 3 · TinyBot v2  🔴 ADVANCED

🤖 TinyBot v2 — Dict-Driven Intent Classifier

🔴 Advanced · Functions · Nested dicts · Set intersection · Runtime dispatch

tinybot_v2.py 
# TinyBot v2 — data-driven, no elif chains (Day 17)
RULES = {
    'greeting': ['hi','hello','hey','greetings'],
    'farewell': ['bye','goodbye','see you'],
    'askname':  ['name','who are you','what are you'],
    'negative': ['sad','tired','upset','unhappy'],
    'positive': ['happy','excited','great','awesome'],
}
REPLIES = {
    'greeting': 'Hi there! 👋',
    'farewell': 'Goodbye! Keep coding! 🚀',
    'askname':  'I am TinyBot v2! 🤖',
    'negative': 'Sorry to hear that. Want help?',
    'positive': 'Awesome! Tell me more. 😊',
    'unknown':  'Hmm, I do not know that yet. 🤔',
}
missed_log = []

def getintent(msg):
    tokens = set(msg.lower().split())
    for intent, kws in RULES.items():
        if tokens & set(kws):  # set intersection
            return intent
    missed_log.append(msg)
    return 'unknown'

print('TinyBot v2 ready. Type bye to quit.\n')
while True:
    msg = input('You: ')
    intent = getintent(msg)
    print('Bot:', REPLIES[intent])
    if intent == 'farewell': break
print('Missed log:', missed_log)

🔑 Key Concepts

  • tokens & set(kws) is set intersection — fires when any keyword matches
  • Separation of concerns: RULES and REPLIES are separate — a content editor can update replies without touching logic
  • missed_log prints on exit — reveals what new intents you need to add

💡 Upgrade Hook

Add a 'food' intent with keywords ['pizza','eat','hungry'] and a matching reply. Adding a new skill takes exactly 2 lines — one in RULES, one in REPLIES.

🤔 Think About It

Type "I'm not sad at all". The bot still replies with the negative response — why? What would you need to add before the set-intersection check to fix this negation bug?

🗺️ Upgrade Decision Map

What Could You Build With These Three Together?

Each project teaches one skill. Here's how those skills combine in real AI systems:

🤖 + ⭐ = Personalised Chatbot

Combine TinyBot v2 with RateRec to build a bot that recommends games based on what a user says. When TinyBot detects the "positive" intent, it ranks and suggests the highest-rated game.

🔍 + 🤖 = Visual Q&A Bot

Run EdgeBot on an image, count the number of cells where a value exceeds a threshold. Pass that count as a message to TinyBot — the bot can then "answer" questions like "does this image have many edges?"

⭐ + 🔍 = Image-Quality Ranker

Use EdgeBot's filtered output as a "sharpness score" for an image. Then use RateRec's sorted() logic to rank multiple images from sharpest (most edges) to blurriest (fewest edges).

🌟 All Three = Mini AI Pipeline

The full loop: EdgeBot extracts features from data → RateRec ranks the results → TinyBot explains the ranking in plain English. That's exactly how many real AI systems work.
